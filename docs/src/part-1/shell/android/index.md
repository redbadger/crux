When we use Crux to build Android apps, the Core API bindings and native
library assets are generated with [BoltFFI](https://www.boltffi.dev/) and
packaged for Gradle to consume.

The shared app types are generated by Crux as Kotlin packages, which we can add
to our Android project using `sourceSets`. The Kotlin code to serialize and
deserialize these types across the boundary is also generated by Crux.
